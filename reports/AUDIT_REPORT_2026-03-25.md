## 6. Main Technical and Trading Risks

### Critical
1. **PAPER_TRADING hardcoded** — typo in code = accidental live trades
2. **No order reconciliation** — partial fills or slow fills = orphaned capital
3. **No E2E integration test** — components tested in isolation but full flow untested
4. **Elite strategy scores possibly overfitted** — 150%+ ROI claims need out-of-sample proof

### High
5. **SL/TP monitor is polling (10s)** — if code crashes between polls, position unprotected
6. **Systemd service incomplete** — no health check, no graceful shutdown
7. **No per-symbol position limit** — could accumulate 30% across 3 copies of same asset

### Medium
8. **Fee model simplified** (fixed 0.1% vs real maker/taker)
9. **No multi-symbol correlation control** — market crash = all positions hit SL
10. **Telegram bot not rate-limited** — user spam could overload CPU
